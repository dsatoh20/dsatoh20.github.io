# High risk clinical characteristics for subarachnoid haemorrhage in patients with acute headache: prospective cohort study
DOI: 10.1136/bmj.c5204
>**臨床医のための疫学・統計 輪読会メモ**

## RQ
神経症状に乏しい頭痛患者から、くも膜下出血（Sabarachnoid haemorrhage; SAH）ハイリスク群を抽出するための臨床的特徴は何か？
## 手法
カナダの5施設共同前向きコホート研究

|||
|--|--|
|P|「1時間以内に痛みのピークに達した非外傷性頭痛、または頭痛を伴う失神を主訴とする」かつ「1時間以内に痛みのピークに達した非外傷性頭痛、または頭痛を伴う失神を主訴とする患者」|
|E|40歳以上、頸部痛・項部硬直、意識消失の目撃、労作時の発症、救急車で搬送、嘔吐、血圧上昇など|
|C|Eの高リスク所見を認めない患者|
|O|SAHであるリスクが高い|

## 知見
- 40歳以上、頸部痛・項部硬直、意識消失の目撃、労作時の発症、救急車で搬送、嘔吐、血圧上昇のうちいずれも認めない場合、SAHであるリスクは低い。
## 考察
- 問診・身体診察だけである疾患を除外診断するルールをつくるとき、条件として、①感度100％を目指す
②誰が使っても同じように評価できる項目を選ぶ③
できるだけ項目は少なく簡便にする3つが挙がると考える。
- ①は、見逃しを許さない医療の現場へ適用するために必要となる。ただし、研究で得た感度100%は内的妥当性に過ぎない。モデルの過適合の可能性を鑑みて、外的妥当性を別途評価すべきである。Ottawa SAH Rulesは、前向き研究にて検証したものであり、ある程度の外的妥当性が担保されている。ただし、実際の現場で自分が用いる際には、より厳密な外的妥当性の評価が必要。
- 感度の高い2値分類モデルをつくるとき、logistic modelよりも、decision tree（classification tree）が適している。classification tree、再帰分割法のよさは、現場でより簡便に適用できることにもある。
- ②身体診察は手技への熟達度や主観的な解釈で結果が揺らいでしまう。ある身体診察項目の結果の解釈が、医療者の間でどれだけ一致するかを表す指標として、κ係数がある。おおよそ、κ > 0.6でその項目を普遍的なものと判断する（マクギーのフィジカル診断学より）。


|術者B\術者A|Positive|Negative|Total|
|--|--|--|--|
|Positive|a|b|a+b|
|Negative|c|d|c+d|
|Total|a+c|b+d|N|

`一致度 = (a + d) / N`

術者Aと術者Bの陽性判断が全く独立と仮定したときの一致率を、偶然一致率と定義すると、

`偶然陽性一致率 = {(a + c) / N} x {(a + b) / N}`

`偶然陰性一致率 = {(b + d) / N} x {(c + d) / N}`

κ係数は、実際の非偶然一致率と期待する非偶然一致率の比だから、

`κ = (一致度 - 偶然一致率) / (1 - 偶然一致率)`


- ③項目を少なく、かつ簡便に結果を得られる項目を選ぶことは、現場で運用するには不可欠な要素。


## 批判的吟味（疫学的視点）by Claude Code

### 1. 研究デザインの妥当性：CDR開発研究としての位置づけ

臨床判断ルール（Clinical Decision Rule; CDR）の開発研究は、McGinn et al.（JAMA 2000）の評価枠組みに従い「導出（derivation）→ 検証（validation）→ 影響分析（impact analysis）」の3段階で評価される。Perry 2010は多施設前向きコホートという点で、単施設後ろ向き導出研究より強いデザインだが、本論文の位置づけ（derivationかvalidationか）を確認しておく必要がある。多施設設計は内的妥当性と外部代表性を同時に高めるが、データ収集の標準化が不十分だと施設間差がかえってノイズになりうる。

### 2. バイアスの検討

**スペクトラムバイアス（spectrum bias）**
研究対象は「救急受診した雷鳴頭痛患者」であり、プライマリケアや市中の頭痛患者とは疾患スペクトラムが異なる。本ルールの適用可能域は救急外来に限定され、一般外来や電話トリアージへの転用は外的妥当性の観点から根拠がない。日本の救急外来への適用にもpretest probability（SAH有病率）の差を考慮する必要がある。

**検証バイアス（verification bias / workup bias）**
参照基準（gold standard）であるCT＋腰椎穿刺が、全対象者に一貫して施行されていたかが重要。低リスク例に腰椎穿刺が省略されていた場合、SAH見落とし例が「陰性」に分類され、感度が過大評価される。本研究がこのバイアスをどう制御しているか（追跡調査による見逃し確認など）を原文で確認することが望ましい。

**情報バイアス（observer variability）**
ユーザーが考察で触れたκ係数の問題点はここに直結する。頸部痛・項部硬直などの身体所見は主観的解釈の余地が大きく、観察者間一致度が低ければルールの再現性は損なわれる。ただしκ係数自体も注意が必要で、Landis & Koch（1977）の解釈ではκ 0.61–0.80は"substantial agreement"だが、κはprevalenceの影響を受けるため、SAHのような低有病率疾患では陽性一致率が構造的に下がりやすい点も踏まえて解釈すべきである。

### 3. 統計手法への補足

**「感度最大化にはClassification treeが有利」という考察の修正**
ユーザーはlogistic回帰よりclassification treeの方が感度を高めるのに適していると述べているが、これは必ずしも正確ではない。Logistic回帰でもカットオフ値（閾値）を操作することで感度を実質100%に設定できる（その分特異度は下がる）。Classification treeの本質的な優位性は「感度の高さ」ではなく**「結果の解釈容易性と臨床運用の簡便さ」**にある——分岐ルールとして視覚化でき、現場スタッフが暗記・即座に適用しやすい。CDR研究でclassification treeが好まれる理由はここにある。

**過適合（overfitting）リスク**
Classification treeはlogistic回帰と比較してoverfittingのリスクが高く、特にサンプルサイズが小さい場合にvalidation datasetで性能が大幅に劣化することがある（optimism bias）。多施設前向き設計はこのリスクを軽減するが、外部検証コホートでの追試が理想的である。

**「all-or-none」ルールの構造的意味**
Ottawa SAH Rulesの「7項目のうち1つでも陽性なら高リスク」という論理積ルール（AND条件の否定）は、構造的に感度を最大化する設計である。項目を増やすほど感度は上がるが特異度は下がる——この感度・特異度トレードオフを明示的に論じることが、CDRの臨床実装評価には不可欠である。

### 4. 内的妥当性と外的妥当性

ユーザーの考察（内的妥当性100%・外的妥当性の別途評価）は適切。補足として、Bayesian視点からは**pretest probabilityの影響が無視できない**。SAHのpretest probabilityが低い集団では、感度が高くても陽性的中率（PPV）が大幅に低下し、「陽性」と出た症例のほとんどが偽陽性になりうる。「感度100%のルールは見逃しを防ぐ」という主張は正しいが、「陽性者全員に精査が必要か」はprevalenceと検査の侵襲性（腰椎穿刺）を含めた意思決定問題として別途論じる必要がある。

## 輪読会(20260628)後の気づき
- 誤差関数で規定されるモデルでは、誤差の絶対値を小さくするように最適化される。そのため、感度100％という極端なモデルに仕上げる余地はない。
- 変数として、2値の項目を選んでいるから、ノンパラメトリックな手法が採用される。
- 感度を最大化するというコンセプトの一方で、Ottawa SAH rule開発に至るに、1時間以内の頭痛--> 雷鳴頭痛の変更がなされた。これは、特異度を高める意図のはずである。感度を保ちながら、特異度を高めることで、より現実的なルールとしての価値を高める狙いが読み取れる。
### 5. 輪読会の気づきへの補足（by Claude Code）

**「誤差関数が感度100%を阻む」という認識の精緻化**

気づきの方向性は正しいが、もう一段深めると正確になる。ロジスティック回帰の損失関数は交差エントロピー（binary cross-entropy）であり、「誤差の絶対値の最小化」（=MAEやMSE）ではなく**対数尤度の最大化**で係数を推定する。いずれにせよ、損失関数はFNとFPを対称的に扱う（クラスウェイト設定をしない限り）ため、「FNをゼロにする＝感度100%」を直接最適化のターゲットにはできない、という洞察は的確である。

ただし正確を期すと：モデルを学習した後に**決定閾値を下げる**（例：P(SAH)≥0.01で陽性とする）ことで感度を事後的に100%近くまで高めることは可能。つまり「ロジスティック回帰では感度100%を達成できない」のではなく、「損失関数の最適化フェーズで直接感度を目標にできないが、閾値操作で補完できる」が正確な整理である。Classification treeは「分岐ルールの構造上、特定の葉ノードを陽性と判定することで感度を直接コントロールしやすい」という実装上の利点がある。

**「2値変数 → ノンパラメトリック手法」という理解の位置づけ**

「予測変数がすべて2値だからclassification treeを使う」という連鎖は実は十分ではない。ロジスティック回帰も2値ダミー変数を問題なく扱えるため、変数の型だけが手法選択の根拠にはならない。

より正確な整理は以下の通り：

| 観点 | ロジスティック回帰 | Classification tree |
|------|-------------------|--------------------|
| 関数形仮定 | 線形ロジット（パラメトリック） | なし（ノンパラメトリック） |
| 交互作用の扱い | 手動指定が必要 | 再帰分割で自動反映 |
| 結果の臨床適用 | オッズ比・確率予測 | YesNo分岐ルール（暗記可能） |
| 感度調整 | 閾値操作で後付け | 分岐条件の選択で直接制御 |

CDR開発でclassification treeが選ばれる本質的な理由は「ノンパラメトリックだから2値変数に向く」ではなく、**「結果がif-thenルールとして直接表現され、現場スタッフが紙1枚で運用できる」**という実装適合性にある。この観点からOttawa SAH Rulesの「7項目のうち1つでもYesなら精査」という形式を評価すると、再帰分割の出力がそのまま臨床プロトコルになっている好例といえる。
